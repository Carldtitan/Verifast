Design a synchronous Moore finite state machine named 'fsm_hold'.
Clock 'clk' and synchronous active-high reset 'rst' are implicit inputs.
Inputs: i0 (1-bit input), i1 (1-bit input), i2 (1-bit input).
Outputs: o0 (1-bit output), o1 (1-bit output).
On reset the machine is in state S0.
It has 3 states: S0, S1, S2.

Per-state behavior (Moore — outputs depend on the current state only):
- In S0: outputs o0=1; o1=1.
    if i1 is 1, go to S0 (checked in this order);
    otherwise go to S0.
- In S1: outputs o0=1; o1=0.
    otherwise go to S2.
- In S2: outputs o0=0; o1=1.
    if i1 is 1, go to S2 (checked in this order);
    if i2 is 1, go to S0 (checked in this order);
    if i2 is 1, go to S2 (checked in this order);
    otherwise go to S2.

Write a single synthesizable SystemVerilog module with this exact interface. Output only the module.
