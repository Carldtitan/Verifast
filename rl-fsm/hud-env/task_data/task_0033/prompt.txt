Design a synchronous Moore finite state machine named 'fsm_fire'.
Clock 'clk' and synchronous active-high reset 'rst' are implicit inputs.
Inputs: i0 (1-bit input), i1 (1-bit input), i2 (1-bit input).
Outputs: o0 (2-bit output).
On reset the machine is in state S0.
It has 6 states: S0, S1, S2, S3, S4, S5.

Per-state behavior (Moore — outputs depend on the current state only):
- In S0: outputs o0=3.
    if i2 is 1, go to S4 (checked in this order);
    otherwise go to S4.
- In S1: outputs o0=2.
    if i0 is 1, go to S1 (checked in this order);
    otherwise go to S5.
- In S2: outputs o0=2.
    if i1 is 1, go to S5 (checked in this order);
    otherwise go to S1.
- In S3: outputs o0=1.
    if i2 is 1, go to S2 (checked in this order);
    if i0 is 1, go to S1 (checked in this order);
    if i1 is 1, go to S3 (checked in this order);
    otherwise go to S4.
- In S4: outputs o0=0.
    if i2 is 1, go to S4 (checked in this order);
    otherwise go to S1.
- In S5: outputs o0=2.
    if i2 is 1, go to S0 (checked in this order);
    if i2 is 1, go to S5 (checked in this order);
    if i2 is 1, go to S0 (checked in this order);
    otherwise go to S5.

Write a single synthesizable SystemVerilog module with this exact interface. Output only the module.
